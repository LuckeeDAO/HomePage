项目术语表（统一术语与用词规范）


> 本表用于规范全项目技术与业务术语，确保跨文档一致性与可引用性。别名/同义词仅用于理解，正文统一按“标准术语”书写。


术语总则

- 标准术语优先使用中文；涉及协议、标准、库名、产品名时保留英文专名（如 WebAssembly、CosmWasm、IPFS、CW20、CW721、Injective Protocol）。

- “承诺/揭示”统一采用“揭示”，不使用“揭秘”。

- “链上最小化 + IPFS主要存储”作为统一的存储边界表述。


技术与架构

- 比特承诺（Bit Commitment）

- 定义：参与者先提交承诺哈希以隐藏真实值，在揭示阶段公开原文与随机性以供验证。

- 别名：承诺方案、承诺协议

- 揭示（Reveal）

- 定义：承诺阶段后的公开原文与随机性以验证承诺的过程。

- 别名：揭秘（不再使用，统一为“揭示”）

- WebAssembly（WASM）

- 定义：面向高性能与可移植的二进制指令格式，用于服务器计算模块。

- Rust / CosmWasm

- 定义：智能合约语言与框架栈（Cosmos 生态）。

- Injective Protocol

- 定义：部署合约与交互的区块链平台（Cosmos 生态）。

- IPFS（InterPlanetary File System）

- 定义：去中心化内容寻址存储网络；本项目的主要明细存储。

- CID（Content Identifier）

- 定义：IPFS内容寻址标识，作为链上校验与检索指针。

- 单一真实源（Single Source of Truth, SoT）

- 定义：以 IPFS 的 `CID/版本` 作为配置与明细数据的唯一真实源；链上仅保留可验证的摘要/哈希/CID与必要索引。

- Merkle 树（Merkle Tree）

- 定义：用于完整性校验的哈希树结构。


代币与NFT

- $LUCKEE 代币

- 定义：权限控制与门槛校验所用的 CW20 代币。

- CW20 / CW721 标准

- 定义：CosmWasm 代币与 NFT 标准。

- NFT 类型（NFT Type）

- 定义：以“类型”抽象承载不同应用场景与全局参数配置的单位。

- 投票项目（NFT属性接口）

- 定义：投票/抽奖项目不是独立模块，而是 NFT 管理的一个属性接口；其配置作为“对应 NFT 类型的全局参数”，以 `CID/版本` 引用；链上仅存结果摘要与必要索引。

- NFT 全局状态变量（Global State of NFT Type）

- 定义：与 NFT 类型关联的跨会话全局状态，用于驱动承诺/揭示等权限与阶段控制。

- 投票资格 NFT（Voting Eligibility NFT）

- 定义：以账户-NFT 持有关系确认投票资格与会话参与权。


决策与流程

- 去中心化决策投票（Decentralized Decision Voting）

- 定义：本项目对“去中心化”的定义由五大原则构成：分散与平等、全员一致性共识、抗合谋与抗操纵、偏好显示自由性、透明与可验证；任何结果由全体参与者偏好叠加产生，不依赖个人或子集，且协议阻止任何规模子集主导或篡改结果。

- 别名：一致同意制决策、强共识决策

- 决策投票（Decision Voting）/ 偏好投票（Preference Vote）

- 定义：参与者提交整数偏好并经统一流程汇总计算结果。

- NFT 类型控制（NFT Type–Driven Control）

- 定义：不同应用领域/决策方式由不同的 NFT 类型承载与控制；类型决定全局参数、状态变量与规则。


- **决策类型分类（按结果基数）**：

- **单目标决策（Single Target Decision）**：从n个候选人中选择1个中奖人，每次仅产生一个获胜结果。

- 实例：单一中奖抽奖、单提案通过投票、唯一获奖者评选

- 特点：结果唯一，竞争激烈，适用于需要明确单一获胜者的场景

- **多目标决策（Multi-Target Decision）**：从n个候选人中选择k个中奖人，奖励相同，一次产生多个互不冲突的获胜结果。

- 实例：多人中奖抽奖（如从100人中选出10个获奖者）、多席位选举、批量奖励分配

- 特点：多个获胜者，奖励均等，适用于需要批量选择获胜者的场景

- **多等级决策（Multi-Tier Decision）**：从n个候选人中选择k组中奖人，每组内部奖励相同，不同组之间奖励不同。

- 实例：等级化抽奖（一等奖1名、二等奖3名、三等奖5名）、分级奖励系统、多层级评选

- 特点：分层奖励，激励差异化，适用于需要建立奖励梯度的场景


- **决策类型分类（按资源属性）**：

- **竞用性资源决策（Competitive Resource Decision）**：资源有限且排他（如名额/额度），需避免冲突与重叠。

- **非竞用性资源决策（Non-Competitive Resource Decision）**：资源非排他，可并行满足（如公共品偏好表达）。


- **多目标选择与偏好值组（Multi-Target Selection & Preference Sets）**

- 定义：n 选 k 的公平选择机制，支持单目标（k=1）、多目标（k>1）和多等级（多组k值）选择。

- 偏好值映射：

- 单目标选择：对应 1 组偏好值（用于计算 1 个结果）。

- 多目标选择：对应 1 组偏好值（用于计算 k 个结果）。

- 多等级选择：对应多组偏好值（各等级各一组，每组计算对应的k个结果）。

- 层级：不作为独立 NFT 类型，而是抽奖/彩票决策类型下的选择模式。


- **统一 NFT 决策状态机**

- 定义：Draft → CommitOpen → CommitClosed → RevealOpen → Revealed → Tallied → Executed → Archived。

- **多等级选择（Tiered Selection）/ 结果序号（Winning Number）**

- 定义：基于配置的公平选择与唯一序号分配机制；当分层存在时等价"多组多目标选择"。

- **多目标选择（n 选 k）**

- 定义：从 n 个参与者中公平选出 k 个不同序号，含基准与偏移计算与防冲突机制。


算法与数学符号

- Φ 映射（Normalization Φ）

- 定义：Φ(x,n)=((x mod n)+n) mod n，输出范围 0..n-1；显示编号采用 1..n（0→n）。

- nextUnique（顺序下延的唯一化算子）

- 定义：在环 0..n-1 上，从起点 start 出发，按“仅在命中未占用时计步”的策略前进 steps 次；若落点仍占用则以步长1继续前进至未占用索引；返回索引。

- 列和（S_j）与偏移（O_j）

- 定义：S_j=Σ_i v_{ij}；O_j=Φ(S_j,n)，j≥2；基准 A_1_idx=Φ(S_1,n)。

- 域分离与盐（Domain Separation & Salt）

- 定义：对每列 j 使用独立域标签 domain_j 与独立盐 salt_j；S_j 以 H(domain_j || v_{ij} || salt_j || ctx) 聚合；ctx 包含会话ID、NFT类型、配置CID、区块高度等。

- 权重快照（Weight Snapshot）

- 定义：在区块高度 h 对账户权重（代币余额/NFT权重）取快照用于计票，避免投票期间转移操纵；配置中记录 snapshotHeight。


资源决策相关

- 竞用性资源决策

- 定义：有限且排他的资源匹配/分配问题（最大权匹配/匈牙利/流模型），满足资格与容量约束。

- 非竞用性资源决策

- 定义：非排他公共品的偏好聚合与阈值/预算内选择问题（阈值通过/背包选择）。


存储边界

- 链上最小记录集（On-chain Minimal Set）

- 定义：账户-NFT 关联、结果摘要/哈希/CID、必要索引与事件。

- IPFS 主要存储（Off-chain via IPFS）

- 定义：承诺/揭示明细、参与者列表、计算过程与证明、完整配置与版本、NFT 元数据。


自动化

- iAgent 自动化（iAgent Automation）

- 定义：对承诺与揭示流程进行策略化、条件化与批量化的自动执行组件。


用词统一与别名规范

- 揭示 = 标准；揭秘 = 别名（不使用于新文本）

- NFT 类型 = 标准；NFT种类 = 别名

- 投票项目属性接口 = 标准；投票项目管理模块（独立）= 过时说法（应表述为 NFT 管理的属性接口）

- 链上最小化 + IPFS主要存储 = 标准；链下存储 = 过时表述（避免）

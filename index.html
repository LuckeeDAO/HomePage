核心技术特点

🔐 比特承诺协议

承诺：c = H(m || r)，其中 m 为偏好/随机数，r 为随机盐
揭示：公开 m 与 r，任何人可复算 H(m || r) 与 c 校验一致
特性：绑定性与隐藏性，保证揭示前隐私、揭示后可验证

🎨 NFT类型驱动架构

NFT类型承载"应用场景配置"的抽象（抽奖/彩票/分配/治理等）
投票/抽奖"项目"是NFT管理的属性接口
资格以"账户-NFT持有"判定，支持转移和交易

⚡ 高性能计算

Rust WebAssembly并行化处理
批处理与缓存策略
支持大规模并发投票和实时结果计算

🔗 去中心化存储

IPFS保存明细，链上仅保存摘要/索引
Merkle/哈希验证链路
单一真实源（SoT）以IPFS的CID作为唯一真实源

示例：去中心化抽奖流程
// 参与者提交随机数 const participants = [3, 7, 2, 9, 1];
// 计算总和 const total = participants.reduce((sum, num) => sum + num, 0); // 22
// 取余得到结果 const result = total % participants.length; // 22 % 5 = 2
// 2号参与者中奖

// 比特承诺示例 const commitment = sha256(preference + randomness);
// 揭示时验证 const isValid = sha256(revealedPreference + revealedRandomness) === commitment;
